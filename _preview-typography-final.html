Co se vlastně mění

Po tvém pozorování ("A breakdown čitelnější, D big čitelnější") jsem zúžil debate na 3 varianty které leží blízko sebe a všechny zachovávají Montserrat pro big číslo. Žádný JetBrains Mono na výsledcích — brand 100% zachován.

CSS diff (Phase 49b skutečný produkční zásah)

.out .big { display: block; font-family: var(--font-brand); font-weight: 500; font-size: clamp(34px, 5.5vw, 56px); line-height: 1; + font-variant-numeric: tabular-nums lining-nums; + font-feature-settings: "tnum" 1, "lnum" 1; margin-bottom: 12px; } .out .breakdown { display: block; font-family: var(--font-ui); font-size: 19px; font-weight: 400; color: var(--ink); + font-variant-numeric: tabular-nums lining-nums; }

Celkem 3 řádky CSS. Žádný HTML zásah, žádný JS. Aplikuje se automaticky na všech 11 tool pages.

Co to fakticky řeší

1. Stabilita pro Phase 51 (live calc): až budeme přidávat live calculation, cifry se nebudou pohybovat při každém úhozu. To je psychologicky důležité — uživatel vidí výsledek měnit hodnotu, ne layout.

2. Vyrovnání v breakdown řádcích: až budou breakdowny obsahovat víc čísel pod sebou (např. Loan: 60 měsíčních splátek), budou pravidelně zarovnané jako tabulka. Profesionálnější dojem.

3. Žádné jiné změny: velikost, váha, rodina písma, padding, border — vše zůstává. Změna je matematická, ne estetická.

Co to NEřeší (a proč to OK)

Big číslo na 56px nebude radikálně jiné než dnes. Tabular-nums u Montserrat dělá cifry trochu širší (kolem "1" je víc bílého místa, "8" se nemění), ale efekt je jemný. Pokud chceš dramatičtější posun (Mono pro big), to je varianta C/B z minulého preview a Tvoje vlastní oči řekly že je to horší pro starší uživatele.

Pravděpodobnost úspěchu (můj odhad)

A current: nikdo se nerozzlobí, ale Phase 51 (live calc) bude trpět nestabilitou cifer.

A+ / D refined: nikdo si nevšimne v statickém stavu, ale Phase 51 bude působit jako přístroj. Doporučuji.
